# Security Policy

## Supported Versions

WiFiSnipe is currently maintained on the latest release branch. Security fixes and updates are provided only for the most recent version available in this repository.

| Version        | Supported |
| -------------- | --------- |
| Latest         | ✅         |
| Older Releases | ❌         |

## Reporting a Vulnerability

The WiFiSnipe project takes security issues seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

Please **do not create a public GitHub issue** for security vulnerabilities.

Instead, report vulnerabilities through one of the following methods:

* GitHub Security Advisories (preferred)
* GitHub private vulnerability reporting
* Contact the maintainer directly through GitHub

### What to Include

When reporting a vulnerability, please provide:

* A detailed description of the issue
* Steps to reproduce the vulnerability
* Impact assessment
* Proof-of-concept (if applicable)
* Screenshots, logs, or relevant output

### Response Timeline

You can expect:

* Initial acknowledgment within **7 days**
* Status updates as investigation progresses
* A coordinated disclosure process when applicable

### Responsible Disclosure

Please allow reasonable time for the issue to be investigated and resolved before publicly disclosing details.

### Scope

WiFiSnipe is a Bash-based wireless auditing and educational security tool. Security reports are particularly welcome for:

* Command injection vulnerabilities
* Privilege escalation issues
* Unsafe file handling
* Insecure temporary file creation
* Dependency-related security concerns
* Arbitrary code execution paths

Reports involving misuse of the tool against unauthorized networks are outside the scope of this policy.

### Recognition

Responsible disclosures that help improve the security of WiFiSnipe will be acknowledged unless anonymity is requested.
